# Docs / Workflows / Threat Research
This directory contains daily threat intelligence research for Elencho.
Each file is a dated report generated by the daily cron job.

## Purpose
- Scrape OSV.dev, npm/PyPI advisories, GitHub Security Advisories for supply-chain threats
- Identify new malware patterns, obfuscation techniques, and attack vectors
- Suggest candidate detection rules for the next coding session
- **Read-only** — never modifies code or rules automatically

## Security Model
1. The cron job produces research reports only — no code changes
2. All rule updates go through the Ed25519-signed manifest pipeline
3. The `make sign` command must be run manually to ship updates
4. Inline suppression allows users to override false positives without waiting for updates

## Workflow
1. Cron job runs daily → produces `YYYY-MM-DD.md`
2. Next coding session: review the latest report, decide which candidate rules to implement
3. Implement new rules → `make sign` → `git tag` → `gh release create`
4. Users get updates via auto-update or `brew upgrade`
